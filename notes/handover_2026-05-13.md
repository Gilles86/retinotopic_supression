## Critical: the NIfTI dtype trap (most important thing to internalize)

**Bug:** `nilearn.maskers.NiftiMasker.inverse_transform(values).to_filename(path)` inherits the **mask's dtype** (uint8 for fmriprep BOLD masks). Nibabel then auto-picks `scl_slope/scl_inter` to fit float values into [0, 255]. Result: **only 256 distinct floats** in the entire brain. One outlier crushes everything around the median into a single bin.

We hit this on m1 PRF amplitude: had 30 unique values across 376k voxels.

**Always wrap when writing a derived parameter NIfTI:**
```python
img = masker.inverse_transform(values)
img.set_data_dtype(np.float32)
img.header.set_slope_inter(slope=1, inter=0)
img.to_filename(path)
```

**Diagnose any new pipeline's output by reading back:**
```python
print(nib.load(p).get_data_dtype())                          # expect float32
print(len(np.unique(np.round(nib.load(p).get_fdata(), 3))))  # expect thousands
```

Documented in `~/.claude/CLAUDE.md` and `retsupp/CLAUDE.md` (§"NIfTI dtype trap").

## Cluster operational lessons

1. **L4-only constraint** (`--constraint=L4` in `fit_prf_l4_chunked.sh`) prevents the cuInit-race that hit us on V100 multi-GPU nodes (`u24-chiivm0-60[3-8]` etc.). 8 jobs on the same 8-GPU node simultaneously calling cuInit deadlocks the NVIDIA driver → TF falls back to CPU → 60× slowdown → walltime timeouts.
   - L4 nodes are single-GPU per node — race impossible by construction.
   - We have 16 L4 nodes total (throttles concurrency but each task is reliable).

2. **30s random sleep stagger** added to chunked-fit script body. Defense-in-depth against both cuInit dogpile (if L4 constraint relaxed someday) and NFS profile-read dogpile.

3. **Lowprio partition** (`--partition=lowprio`) dispatches in seconds instead of waiting hours behind fairshare. Same physical L4/A100/V100 nodes. Preemption risk for 15-30 min chunks is low. **All AF + chain scripts now default to lowprio.**

4. **Per-subject dep chains** (not phase-wide afterok). One sub-N's failure only kills sub-N's downstream. See `submit_prf_sweep_persub.sh`.

5. **DN model needs CUDA 12** for H100/H200 (sm_90). We're on CUDA 11.8. Building a `retsupp_cuda12` env is a future task.
